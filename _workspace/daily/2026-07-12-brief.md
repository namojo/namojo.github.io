# 팩트 카드 — 2026-07-12 (KST)

## 토픽
**JADEPUFFER — 최초로 문서화된 '에이전틱 랜섬웨어'**: LLM 에이전트가 정찰부터 자격증명 탈취·측면 이동·권한 상승·데이터베이스 암호화·협박까지 공격 전 과정을 사람 개입 없이 자율 수행한 첫 사례.

## 핵심 팩트
- **발견/공개 주체:** 시스디그Sysdig 위협연구팀(Threat Research Team, TRT). "assessed to be the first documented case of agentic ransomware."
- **공개 시점:** 2026년 7월 초 (Forbes 7/7, 다수 매체 7/8~7/10 보도). → 글 시점은 "이번 주", "7월 초 시스디그가 공개한"으로 표기.
- **초기 침투 경로:** CVE-2025-3248 — Langflow(오픈소스 AI 앱/에이전트 워크플로 구축 도구)의 인증 누락(missing-authentication) RCE 취약점. 로그인 없이 서버에서 임의 파이썬 코드 실행. **Langflow 1.3.0에서 패치**, **2025년 5월 CISA KEV(알려진 악용 취약점) 목록 등재**.
- **공격 체인(전 과정 자율):** 정찰 → 자격증명 스윕 → 측면 이동 → 권한 상승 → DB 협박·암호화.
  - 탈취 대상 자격증명: AI 서비스 API 키(OpenAI, Anthropic, DeepSeek, Gemini), 클라우드 자격증명(알리바바·텐센트 등 중국 제공사 + AWS·GCP·Azure), 암호화폐 지갑 키, DB 로그인.
- **핵심 수치:**
  - Nacos 서비스 설정 항목 **1,342개**를 암호화한 뒤 원본 삭제.
  - 로그인 실패 → 다단계 정정까지 **31초**(자율 오류 복구 정황).
- **LLM 판별 근거(자율성 정황):**
  - 페이로드에 자연어 주석이 가득 — 각 행동의 이유, 타깃의 ROI 우선순위("largest" DB 식별), 단계별 목적 설명. 사람 오퍼레이터는 잘 안 쓰지만 LLM 생성 코드가 반사적으로 남기는 주석.
  - 실패 후 빠른 자율 복구.
- **어떤 LLM인지:** 특정 모델(GPT/Claude 등) 미확정. 시스디그는 "system prompt/agent configuration에 대한 가시성이 없다"고 명시.
- **랜섬 노트:** 파일명 README_RANSOM. 비트코인 주소는 공개 문서에서 널리 쓰이는 '예시 주소'(3J98t1Wp...) → LLM이 학습 데이터에서 그대로 재현했을 가능성. 구체적 금액 미명기.
- **의미(전문가 코멘트 종합):**
  - "랜섬웨어 실행의 기술 문턱(skill floor)이 '에이전트를 돌리는 비용' 수준으로 떨어졌다. LLMjacking(탈취 API 키)으로 돌리면 공격자 비용은 0에 가깝다."
  - "방어 모델은 상대편에 '사람'이 있다고 가정하는데, 상대가 에이전트가 되는 순간 그 산수가 무너진다. 어떤 분석가 팀도 에이전트가 행동하는 속도보다 빠르게 분류할 수 없다."
  - '에이전틱 SOC'(자율 방어) 개념 부상. CrowdStrike Charlotte AI: 수동 조사 업무 약 70% 절감, 판단 정확도 98% 초과 주장.

## 평론 각도 (namojo 문체)
- 이 블로그는 최근 '에이전트가 일을 대신하는' 흐름(챗GPT 워크 슈퍼앱, 토큰맥싱→효율)을 연재해 왔다. JADEPUFFER는 **바로 그 에이전틱 능력의 뒷면** — 같은 자율성이 방어가 아니라 공격에 붙었을 때. "일을 대신하는 손"이 "공격을 대신하는 손"이 된 순간.
- 역사 비유 후보: 자동화의 양면(방직기가 생산도, 실직도 가져왔듯) / 불(火)의 이중성 / 다이너마이트(노벨) — 도구는 의도를 묻지 않는다.
- 한국적 맥락: Langflow·Nacos는 한국 AI 스타트업·MSA 스택에서 흔히 쓰임. 인증 없이 열린 내부 AI 도구, 방치된 API 키(LLMjacking) — 한국 SOC/스타트업의 현실 위험. 방어도 자동화(에이전틱 SOC)로 맞서야 하는 '속도 대칭' 문제.
- 비판적 균형: 과장 경계 — 침투 경로 자체는 알려진 취약점(패치 존재)이고 정교함이 새로운 게 아니라 '자율 실행'이 새롭다. 종말론이 아니라 위생(패치·최소권한·키 관리)의 재확인.
- 마무리: 질문형 — 우리 조직의 열린 AI 도구와 방치된 키는 지금 누구의 손이 잡을 수 있는가.
